# UPI Payment Tracker

This local tool imports successful UPI credits into ERPNext `Bank Transaction`
records and installs the `UPI Payments Today` report for daily tracking.

Google Pay is not used as the source for all incoming UPI credits. Google Pay
merchant APIs can initiate or check Google Pay payment flows, but Google Pay
transaction history is limited to Google Pay transactions, not every UPI or bank
transaction. For full daily UPI tracking, use the bank, PSP, or payment-gateway
settlement feed.

## Setup

List the ERPNext bank accounts and choose the account that receives UPI credits:

```powershell
$env:ERPNEXT_API_KEY="your_key"
$env:ERPNEXT_API_SECRET="your_secret"
python upi_payment_tracker.py list-bank-accounts
```

Set the selected bank account:

```powershell
$env:UPI_BANK_ACCOUNT="HDFC Bank - HDFC Bank"
```

Install or update the ERPNext report:

```powershell
python upi_payment_tracker.py install-report
```

Open the report:

```text
http://localhost:8080/app/query-report/UPI%20Payments%20Today
```

## Daily Import

Export the day's successful UPI credits from your bank, PSP, or payment-gateway
dashboard and map it to this CSV format:

```text
date,amount,utr,payer_vpa,payer_name,description,status
2026-06-08,1250.50,612345678901,customer@upi,Customer Name,Counter QR,SUCCESS
```

Run a dry run first:

```powershell
python upi_payment_tracker.py import-csv .\upi_payments_template.csv --dry-run
```

If the dry run looks correct, import for reconciliation:

```powershell
python upi_payment_tracker.py import-csv .\upi_payments_template.csv
```

The importer creates submitted `Bank Transaction` records with:

- `transaction_type = UPI`
- `transaction_id = UPI:<utr>`
- `reference_number = <utr>`
- `status = Unreconciled`

It skips failed or pending payments and avoids duplicates by checking the UPI
transaction id first, then the fallback combination of UTR, date, amount, and
bank account.

## Google Pay Omnichannel Import

If you are onboarded for Google Pay Omnichannel API, you can check Google Pay
merchant transactions by merchant transaction id and import successful UPI
payments into the same ERPNext reconciliation flow.

Required Google setup:

- `GOOGLE_PAY_MERCHANT_ID`
- `GOOGLE_APPLICATION_CREDENTIALS`, pointing to the service-account JSON file
- Google allowlisting for the Omnichannel API service account
- Merchant transaction IDs generated by your checkout/payment request flow

Prepare a CSV of merchant transaction IDs:

```text
merchant_transaction_id,description
ORDER-2026-0001,Counter QR request
```

Run a dry run:

```powershell
$env:GOOGLE_PAY_MERCHANT_ID="your_google_merchant_id"
$env:GOOGLE_APPLICATION_CREDENTIALS="D:\secure\google-service-account.json"
python upi_payment_tracker.py import-google-transactions .\google_pay_transactions_template.csv --dry-run
```

Then import:

```powershell
python upi_payment_tracker.py import-google-transactions .\google_pay_transactions_template.csv
```

This command calls Google's Omnichannel `merchantTransactions:get` API for each
merchant transaction ID. It imports only `SUCCESS` UPI transactions and uses the
Google `upiRrn` as the ERPNext UTR/reference number.

Google does not provide a general "list every UPI credit for the day" feed from
Google Pay transaction history. For all UPI credits across Google Pay, PhonePe,
Paytm, bank apps, and other UPI apps, the bank/PSP/payment-gateway feed remains
the source of truth.

## Provider Notes

The CSV provider is still the default because no live merchant/bank API
credentials are configured in this workspace. The Google provider supports
transaction-detail checks for onboarded Google Pay Omnichannel merchants; it
does not replace a bank settlement feed for all incoming UPI credits.

Official references:

- Google Pay India Merchant SDK: https://developers.google.com/pay/india/api/merchant-sdk/guides/overview
- Google Pay India Omnichannel API: https://developers.google.com/pay/india/api/otherapis/omnichannel/overview
- Google Pay Omnichannel authorization: https://developers.google.com/pay/india/api/otherapis/omnichannel/authorization
- Google Pay Get Transaction Details: https://developers.google.com/pay/india/api/otherapis/omnichannel/get-transaction-details
- Google Pay transaction history limitation: https://support.google.com/pay/india/answer/7430307?hl=en
